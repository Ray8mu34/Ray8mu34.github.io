1. Git安装

1.1 Git简介

Git是一种分布式版本控制系统

在实际应用中，一项工作很可能需要长期、多次的修改才能完成，为了保存不同时期的版本，我们需要为每一次修改新建副本。最后一项工作下会有很多个文件夹

而Git提供了一种管理这些不同版本的方法，在这里无须了解其工作原理，只须大致明白其工作区域：

  • Workspace：工作区
  • Stage：暂存区
  • Repository：仓库（通常在本地）
  • Remote：远程仓库

（远程）仓库即是各种版本信息保存的地方。

作为分布式系统，Git使得每个人的电脑都有项目的完整版本库。

浅谈Python环境

浅谈python环境

1. Python环境包含什么

一般来说，一个python环境包含以下三个要素：

  • Python解释器
  • Python库（标准库与第三方库）
  • 可执行程序的位置

由于标准库在下载解释器时往往自带，所以我们关注的其实是自己所需的第三方库，管理第三方包的常用工具有pip和conda

而要利用Python进行开发，我们还需要编辑器或IDE

2. 官方Python安装

2.1 安装

Python官方提供了环境的下载，也即是我们熟知的Python 官方网址
在安装时，还额外提供了一款基本IDE（IDLE）和pip工具等选项

除了官方提供的版本，也有一些Python的发行版，包含Python解释器和一部分包，以及其它工具。anaconda即是这样一个例子

2.2 主体

安装好的Python里最主要的文件（夹）有如下几个：

  • python.exe
  • Lib文件夹
  • Scripts文件夹

python.exe即是解释器，在调用时使用该程序解释执行python代码；Lib文件夹，包括其内部的 site-packages，用于存放包；Scripts文件夹用于存放可执行文件和脚本，如下载其它第三方包时用到的pip

一般而言，安装的包都会放在 Lib文件夹下。如果要详细查看包的搜寻路径，则可以使用python内部的变量 sys.path，打印其值

2.3 环境变量

如果没有在安装时选择add python to path，为了让其它应用能够启动Python解释器，我们需要将 python.exe添加到系统的环境变量内；同时要使用各种可执行文件，还需要将 Scripts文件夹也加入环境变量

添加完成后，在命令行输入 python 即可查看Python版本，如果系统内安装了多个版本的Python，则会显示路径在系统环境变量里最靠前的一个

3. 虚拟环境是什么

3.1 为什么需要虚拟环境

如果需要在同一终端开发多个python项目，很可能各项目之间使用的包的版本、python的版本等有所不同，有时候甚至相互冲突，因而，我们需要多个python环境进行开发

所谓的虚拟环境，其实取名叫环境副本或许更合适。选取一个新的文件夹，命名保存，就可以创建一个新的环境。为了方便管理，python标准库提供了 venv 模块用于创建、管理虚拟环境

3.2 虚拟环境的构成

虚拟环境所在的文件夹也包含如下三个部分

  • python.exe
  • Lib文件夹
  • Scripts文件夹

与一般环境不同的是，python.exe是放在 Scripts文件夹下的，所以在添加环境变量时只需要将 Scripts加入

3.3 使用 venv模块创建虚拟环境

通过在命令行输入如下命令可以创建虚拟环境

这样会产生新建文件夹，里面即包含了一个虚拟环境，在 Scripts目录下会有 activate和 deactivate两个文件

在命令行界面使用时，通过 activate/deactivate命令来激活/退出环境，在激活时需要输入activate的完整路径。

所谓激活其实是将Scripts目录临时加入系统环境变量的首位，这样系统优先使用的即是该目录下的python解释器

在IDE和编辑器中使用时，则根据选择的Python解释器的路径，会自动地使用相应目录下的包，也即自动切换了虚拟环境。这也意味着在命令行界面中，只要在相应解释器所在的目录下启动python，不需要激活也能使用虚拟环境——虚拟环境其实就是正常的环境

自己配置环境对经验不足的人来说比较困难，可以考虑使用其它工具辅助，如anaconda提供的包管理工具。在本文不做赘述
